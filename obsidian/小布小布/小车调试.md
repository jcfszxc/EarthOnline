


# 阶段1：硬件系统启动


## 终端1：ROS核心服务



```shell
sh kill_ros.sh
```


**作用：**终止之前的程序。


```shell
cd ~/catkin_ws_clean
source devel/setup.bash
roscore
```


**作用：** ROS主节点，负责节点注册、话题管理、服务发现。所有ROS节点都需要它才能互相通信。**必须第一个启动。**


## 终端2：机器人驱动层



```shell
cd ~/catkin_ws_clean
source devel/setup.bash
roslaunch robot_driver robot_driver.launch
```

**作用：**

- 启动 `robot_driver_node` - 发布机器人状态信息（里程计、电池、超声波等）
- 发布TF坐标变换（雷达、IMU相对于机器人base_link的位置）
- 可能还负责一些高层的数据处理

**发布的话题：** `/odom`, `/battery_info`, `/motor_speed`, `/ultrasonic*`, `/tf`



## 终端3：底层电机控制

```shell
cd ~/catkin_ws_clean
source devel/setup.bash
sudo chmod 666 /dev/ttyS0  # 给串口权限（只需运行一次）
rosrun stm32_node stm32f427_node
```

**作用：**

- 与STM32微控制器通信（通过串口 `/dev/ttyS0`）
- **接收 `/cmd_vel` 话题**，将速度命令转换为电机控制信号
- 读取底层传感器数据并反馈

**这是真正控制电机的节点！** 没有它，发送cmd_vel不会有任何反应。



## 终端4：雷达系统

```shell
cd ~/catkin_ws_clean
source devel/setup.bash
roslaunch robot_description start_devices.launch
```

## 🎯 **使用RViz可视化（强烈推荐）**

RViz可以让你直观地看到导航过程：

```bash
# 新终端
cd ~/catkin_ws_clean
source devel/setup.bash
rviz
```

**在RViz中配置**：

1. Fixed Frame 设为 `map`
2. 添加：
    - Map → /map（地图）
    - LaserScan → /scan（激光）
    - RobotModel（机器人模型）
    - Path → /move_base/NavfnROS/plan（全局路径）
    - Path → /move_base/TebLocalPlannerROS/local_plan（局部路径）
3. **用工具栏的"2D Pose Estimate"设置初始位姿**
4. **用"2D Nav Goal"发送导航目标**
