## 1.2 系统概述

**系统用途**：三光融合与识别处理软件用于海上目标检测识别，通过融合可见光相机、红外相机和激光雷达三种传感器数据，实现对海上船舶等目标的全天候、高精度检测识别。

**源代码特性**：

- **代码规模**：源代码总计约1.5万行，包含12个主要源文件，31个头文件
- **编程语言**：C++14标准，符合ISO/IEC 14882:2014规范
- **架构设计**：采用模块化设计，支持多线程并行处理
- **第三方依赖**：集成OpenCV、GStreamer、RKNN等主流库
- **平台适配**：支持ARM64架构，针对RK3588平台优化

**开发环境与工具链**：

- **开发平台**：Ubuntu 18.04 LTS，支持交叉编译
- **编译器**：GCC 7.5.0，支持C++14特性
- **构建系统**：CMake 3.16，支持多平台构建
- **版本控制**：Git版本管理，支持分支开发

# 5 配置参数说明

## 5.1 配置文件结构

配置文件采用JSON格式，包含十一个主要配置段：

```json
{
  "default_blend_type": "VI",           /* 默认融合类型 */
  "DEBUG_OPTIONS": { /* 调试选项配置段 */ },
  "network_settings": { /* 网络配置段 */ },
  "video_devices": { /* 视频设备配置段 */ },
  "visible_camera_settings": { /* 可见光相机配置段 */ },
  "infrared_camera_settings": { /* 红外相机配置段 */ },
  "image_dimensions": { /* 图像尺寸配置段 */ },
  "ir_settings": { /* 红外处理配置段 */ },
  "video_settings": { /* 视频编码配置段 */ },
  "rknn_settings": { /* 推理引擎配置段 */ },
  "yolo_settings": { /* 目标检测配置段 */ },
  "calibration": { /* 相机标定配置段 */ },
  "lidar_settings": { /* 激光雷达配置段 */ }
}
```

## 5.2 系统基础配置

### 5.2.1 默认融合模式

```json
{
  "default_blend_type": "VI"            // 默认图像融合类型："VI"(可见光+红外)、"V"(仅可见光)、"I"(仅红外)
}
```

## 5.3 调试选项配置段（DEBUG_OPTIONS）

```json
{
  "DEBUG_OPTIONS": {
    "strict_detection_mode": true,       // 严格检测模式开关
    "draw_bounding_boxes_locally": true, // 本地绘制检测框开关
    "PRINT_RECEIVED_COMM": true,         // 打印接收通信数据
    "LOG_RECEIVED_COMM": true,           // 记录接收通信日志
    "PRINT_SENT_COMM": false,            // 打印发送通信数据
    "LOG_SENT_COMM": false,              // 记录发送通信日志
    "LOG_VISIBLE_CAMERA_DATA": false,    // 记录可见光相机数据
    "LOG_VISIBLE_CAMERA_STATUS": false,  // 记录可见光相机状态
    "save_images": false,                // 图像保存开关
    "save_interval": 5,                  // 图像保存间隔(秒)
    "status_frame_debug": true,          // 状态帧调试开关
    "status_frame_modifications": {      // 状态帧修改配置
      "6": "00001111",                   // 帧ID 6的状态位设置
      "7": "00000001",                   // 帧ID 7的状态位设置
      "8": "00000111",                   // 帧ID 8的状态位设置
```

## 5.14 配置参数使用说明

### 5.14.1 关键参数调整建议

- **检测阈值**: box_thresh建议根据实际场景调整，室外环境可适当降低至0.2
- **网络配置**: IP地址需根据实际网络环境配置，确保设备间可正常通信
- **相机标定**: 标定参数需通过专业标定工具获得，直接影响融合精度
- **雷达外参**: 外参矩阵需精确测量，误差会导致点云投影偏移

### 5.14.2 调试模式说明

- **开发阶段**建议开启draw_bounding_boxes_locally和相关日志选项
- **生产环境**建议关闭所有调试选项以提升性能
- **save_images选项**仅在数据采集和问题分析时开启

### 5.14.3 性能优化参数

- **initial_bps_rate**: 根据网络带宽调整，避免丢帧
- **YOLO检测参数**: 在精度和速度间平衡
- **图像分辨率**: 可根据计算资源适当调整

# 7 故障诊断与排除

## 7.1 常见故障处理

### 7.1.1 启动失败问题

**库文件依赖问题：**

**故障现象：**

```
./build/VIS_NIR_Radar_Fusion_CCT2143_XMU: error while loading shared libraries: librknn_api.so: cannot open shared object file: No such file or directory
```

**诊断步骤：**

```bash
# 1. 检查库文件是否存在
ls -la runtime/librknn_api/aarch64/librknn_api.so

# 2. 检查库路径配置
echo $LD_LIBRARY_PATH

# 3. 使用ldd检查依赖
ldd build/VIS_NIR_Radar_Fusion_CCT2143_XMU | grep "not found"
```

**解决方案：**

```bash
# 方案1：设置临时库路径
export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:$(pwd)/runtime/librknn_api/aarch64:$(pwd)/runtime/rkrga/lib

# 方案2：永久设置库路径

```

```bash
sudo systemctl restart trimodal-fusion.service
```
