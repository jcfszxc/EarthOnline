# 3 软件综述

## 3.1 软件应用

三光融合与识别处理软件专门设计用于海上目标的自动探测、识别和跟踪。通过整合可见光、红外热成像和激光雷达三种传感器技术，系统能够在各种复杂海洋环境和天气条件下提供可靠的目标识别服务。

**预期用途：**

- 海上船舶的实时探测和自动识别
- 多目标同时跟踪和轨迹分析
- 异常目标的警报和预警
- 海域态势感知和监控

**期望能力提升：**

- **全天候工作能力**：克服传统单一传感器在恶劣天气下的局限性
- **识别精度提升**：相比单一传感器方案，目标识别准确率提升30%以上
- **自动化水平**：减少人工监控工作量，提高监控效率
- **适应性增强**：能够适应不同海况和光照条件

**运行改进效果：**

- 大幅提高海上目标监控的自动化程度
- 显著减少误报和漏报情况
- 提升复杂环境下的目标识别能力
- 为海上安全监管提供技术支撑

# 4 软件入门

## 4.1 软件的首次用户

### 4.1.1 熟悉设备

首次使用三光融合与识别处理软件的用户需要熟悉RK3588计算平台和相关设备的基本操作。

**电源操作规程：**

1. **开机前检查**：确认所有电源线连接牢固，电源指示灯正常
2. **开机顺序**：首先接通外围设备电源（传感器、网络设备），然后启动RK3588主机
3. **电源指示**：正常启动后，主机电源指示灯应显示绿色常亮
4. **关机程序**：使用软件正常退出后，等待系统完全关闭再断开电源

**显示和交互设备：**

由于RK3588计算卡运行在无界面Linux环境下，主要通过以下方式进行交互：

**串口终端连接**：

- 连接参数：115200波特率，8数据位，1停止位，无校验
- 终端软件：推荐使用PuTTY、SecureCRT或minicom
- 连接线缆：使用标准USB转串口线缆

**SSH网络连接**：

- 默认IP地址：192.168.127.100-130（可配置）
- 用户名：root或指定用户名
- 认证方式：密码认证或SSH密钥认证

**LED状态指示：**

- **电源LED**：绿色常亮表示正常，红色表示故障
- **网络LED**：绿色闪烁表示网络活动，橙色表示网络故障
- **运行LED**：蓝色闪烁表示软件正常运行

### 4.1.3 安装和设置

完整的软件安装和配置过程包括系统准备、软件安装、环境配置和功能验证几个步骤。

**安装前准备：**

1. **硬件检查**：确认RK3588计算卡和所有传感器设备工作正常
2. **网络配置**：设置静态IP地址，确保网络连通性
3. **存储空间**：检查可用存储空间，至少需要10GB空闲空间

**系统镜像安装：**

```bash
# 1. 下载系统镜像
wget http://releases.rockchip.cn/rk3588-ubuntu-20.04.img

# 2. 写入SD卡
sudo dd if=rk3588-ubuntu-20.04.img of=/dev/sdX bs=1M

# 3. 启动系统并完成初始化
```

**依赖软件安装：**

```bash
# 更新系统包
sudo apt update && sudo apt upgrade -y

# 安装基础工具
sudo apt install build-essential cmake git wget -y

# 安装OpenCV
sudo apt install libopencv-dev python3-opencv -y

# 安装深度学习框架
pip3 install rknn-toolkit-lite2
```

**软件部署：**

```bash
# 1. 创建安装目录
sudo mkdir -p /opt/three_light_fusion

# 2. 复制程序文件
sudo cp VIS_NIR_Radar_Fusion_CCT2143_XMU /opt/three_light_fusion/
sudo cp config.json /opt/three_light_fusion/
sudo cp *.yaml /opt/three_light_fusion/

# 3. 设置执行权限
sudo chmod +x /opt/three_light_fusion/VIS_NIR_Radar_Fusion_CCT2143_XMU

# 4. 配置环境变量
echo 'export PATH=$PATH:/opt/three_light_fusion' >> ~/.bashrc
source ~/.bashrc
```

**初始配置验证：**

1. **硬件连接测试**：检查所有传感器设备是否正确识别
2. **网络连通性测试**：验证与上位机和传感器的网络连接
3. **功能模块测试**：运行内置的自检程序，验证各功能模块

## 4.2 启动

软件启动是一个自动化程度较高的过程，系统会依次初始化各个模块并进行自检。

**标准启动流程：**

1. **登录系统**

```bash
# SSH方式登录
ssh root@192.168.127.100

# 或使用串口终端登录
```

2. **环境检查**

```bash
# 检查硬件状态
lsusb  # 检查USB设备
ifconfig  # 检查网络接口

```

3. **启动软件**

```bash
# 进入软件目录

# 启动主程序

```

**启动自检过程：**

系统启动时会自动执行以下检查：

|检查项目|检查内容|正常状态|异常处理|

# 5 使用指南

## 5.1 能力

三光融合与识别处理软件为用户提供了全面的海上目标监控和识别能力，各功能模块之间紧密协作，形成完整的处理链路。

**核心处理能力概述：**

**多传感器数据融合**：系统能够同时处理可见光图像、红外热像图和激光雷达点云数据，通过先进的融合算法生成综合的目标信息。融合过程包括时间同步、空间配准和特征级融合，确保不同传感器数据的有效整合。

**智能目标检测**：基于深度学习技术，系统能够自动识别和分类海上目标，包括各类船舶、浮标和其他海上设施。检测算法针对海洋环境特点进行优化，能够有效应对海浪、光照变化和天气影响。

**实时跟踪处理**：系统支持多目标同时跟踪，能够为每个检测到的目标建立运动轨迹，预测运动趋势。跟踪算法具有良好的鲁棒性，能够处理目标遮挡、交叉和暂时消失等复杂情况。

**自适应环境调节**：根据当前的环境条件（光照、天气、海况等），系统能够自动调整处理参数和算法策略，保持最佳的检测性能。

## 5.6 错误、故障和紧急情况下的恢复

系统具备完善的故障检测和恢复机制，能够应对各种异常情况并快速恢复正常运行。

**常见故障类型和恢复方法**

**传感器设备故障**

_症状_：传感器数据中断或异常 _诊断_：检查设备连接状态和设备自检结果 _恢复步骤_：

1. 检查物理连接线缆
2. 重启设备电源
3. 重新初始化设备驱动
4. 如故障持续，切换到其他传感器模式

```bash
# 重启传感器设备
sudo systemctl restart camera_driver
sudo systemctl restart lidar_driver
```

**网络通信故障**

_症状_：网络连接中断或数据传输异常 _诊断_：检查网络接口状态和连接质量 _恢复步骤_：

1. 检查网络线缆连接
2. 重新配置网络接口
3. 重启网络服务
4. 启用离线缓存模式

```bash
# 网络故障诊断和恢复
sudo ifconfig eth0 down
sudo ifconfig eth0 up
sudo systemctl restart networking
```

**软件模块故障**

_症状_：特定功能模块无响应或输出异常 _诊断_：查看模块日志和资源使用情况 _恢复步骤_：

1. 重启故障模块
2. 清理临时文件和缓存
3. 检查内存和存储资源
4. 必要时重启整个软件

```bash
# 软件模块恢复
sudo killall -HUP VIS_NIR_Radar_Fusion_CCT2143_XMU
sudo rm -rf /tmp/three_light_fusion_*
```

**系统资源不足**

_症状_：处理速度明显下降或系统卡顿 _诊断_：监控CPU、内存和存储使用情况 _恢复步骤_：

1. 清理不必要的进程和文件
2. 优化处理参数降低资源需求
3. 重启系统释放资源
4. 检查硬件故障

**紧急恢复程序**

当系统出现严重故障时，执行紧急恢复程序：

**阶段1：故障隔离**

# 6 注释

## 6.1 术语和定义

**三光融合（Three-Light Fusion）**：指同时使用可见光、红外光和激光雷达三种不同光谱传感器进行数据采集和信息融合的技术。

**时空配准（Spatiotemporal Registration）**：将不同传感器在不同时间和空间获取的数据进行对齐和匹配的过程。

**非极大值抑制（Non-Maximum Suppression, NMS）**：目标检测中用于消除重复检测框的算法。

**置信度阈值（Confidence Threshold）**：判断检测结果可靠性的数值标准。

**点云数据（Point Cloud Data）**：激光雷达生成的三维空间点集合，包含位置和反射强度信息。
