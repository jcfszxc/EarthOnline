## 1.2 系统概述

**系统用途**：三光融合与识别处理软件用于海上目标检测识别，通过融合可见光相机、红外相机和激光雷达三种传感器数据，实现对海上船舶等目标的全天候、高精度检测识别。

**一般特性**：

- **规模特性**：软件采用模块化设计，包含数据采集、预处理、融合、检测识别、通信管理、状态控制等6个主要功能模块，代码规模约1.5万行
- **实时性特性**：融合处理频率≥25Hz，识别处理频率≥10Hz，允许误差：±5%
- **可靠性特性**：支持长时间连续运行≥8小时，具有容错处理能力和自动恢复机制
- **智能化特性**：集成深度学习算法，支持≥7类目标类型识别，识别准确率≥85%
- **适应性特性**：支持七种工作模式，适应不同海况和气象条件

**开发、运行和维护历史**：

- **开发历程**：项目于2024年启动，经历需求分析、系统设计、原型开发、系统集成测试等阶段
- **运行历史**：已在实验室环境完成功能验证，在模拟海洋环境完成初步性能测试
- **维护计划**：计划建立技术支持体系，包括软件升级、故障处理、性能优化等服务

**相关机构**：

- **需方**：中国电子科技集团公司第二十七研究所
- **开发方**：厦门大学信息学院

**运行现场**：

- **当前运行现场**：厦门大学信息学院实验室、海洋传感器模拟测试平台
- **计划运行现场**：海上测试平台及试验船舶

## 3.1 核心技术架构决策

基于系统实时性、精度要求和工程可实现性的综合考量，本软件采用模块化、并行化的技术架构，在关键技术路线选择上实现了性能与复杂度的最优平衡：

**通信架构**：采用UDP协议结合JSON配置的轻量化通信方案。UDP协议传输延迟相比TCP协议延迟降低60%，配合JSON格式配置文件，解析效率相比XML格式提升30%以上。

**融合算法**：采用基于加权平均的三光源实时融合技术。使用硬编码外参矩阵实现毫秒级点云-图像配准，权重比例为可见光:红外:激光雷达=0.4:0.4:0.2。

**检测策略**：自研深度学习算法结合多线程并行处理。检测模型采用独立检测线程策略，检测准确率达到≥85%。

**系统架构**：多线程并行结合NPU硬件加速。采用采集、处理、识别、融合四级流水线并行处理模式，NPU负责深度学习推理，CPU承担数据处理。

## 3.3 设计约定

基于上述设计决策，制定以下关键设计约定：

**性能约定**：

- 融合处理频率≥25Hz，允许误差：±5%
- 目标检测频率≥10Hz，允许误差：±5%
- 系统资源使用率：CPU≤80%，内存≤80%

**接口约定**：

- 视频采集统一采用V4L2接口
- 网络通信采用UDP协议，控制端口5000，状态端口5100
- 串口通信波特率统一为115200
- 所有数据传输包含校验信息

**数据处理约定**：

- 图像格式：输入YUV420，输出BGRA
- 检测结果包含类别、位置、置信度、距离四要素
- 状态数据固定134字节格式，25Hz频率发送
- 目标识别准确率≥85%

**传感器规格约定**：

- 可见光相机：1920×1080@25fps，设备节点/dev/video8
- 红外相机：1280×1024@25fps，设备节点/dev/video0
- 激光雷达：10Hz点云数据，UDP端口2368/2369
- 探测距离：可见光≥500m，红外≥300m，激光雷达≥150m

**容错约定**：

- 网络异常时30秒内重连
- 关键操作记录日志，保留7天历史数据

本章所述设计决策和约定为后续的CSCI体系结构设计和详细设计提供指导原则和约束条件。所有设计决策均通过决策标识符进行编码管理，确保在整个开发周期中的可追溯性。

### 4.1.3 每个软件单元的用途和分配的CSCI需求

#### 4.1.3.1 工作流控制模块

**工作流控制模块**（[[XMU-SG-SDD-WFC-001]]）是整个系统的核心，负责协调和控制其他模块的工作。该模块实现了[[XMU-SG-SRS-ModeSwitch]]模式切换能力需求，管理系统的初始化、光源切换、数据融合、目标检测和结果输出等关键流程，确保整个系统的协调运行。

#### 4.1.3.2 数据采集模块

**GStreamer图像采集模块**（[[XMU-SG-SDD-GST-003]]）实现了[[XMU-SG-SRS-VideoProcessing]]视频处理能力需求中的数据采集子能力，利用GStreamer框架实现可见光和红外图像的采集和预处理。该模块对应需求规格说明中的可见光视频采集接口（[[XMU-SG-SRS-IF-HW-001]]）和红外视频采集接口（[[XMU-SG-SRS-IF-HW-002]]）。

**点云数据处理模块**（[[XMU-SG-SDD-PCD-005]]）实现了[[XMU-SG-SRS-LidarRanging]]激光测距能力需求，处理来自激光雷达的点云数据，并与图像数据进行配准。该模块对应激光雷达数据接口（[[XMU-SG-SRS-IF-HW-003]]）需求。

#### 4.1.3.3 数据处理模块

**RGA图像处理模块**（[[XMU-SG-SDD-RGA-004]]）实现了[[XMU-SG-SRS-DataFusion]]数据融合能力需求中的图像融合子能力，利用RGA硬件加速库进行图像格式转换、伪彩色处理和图像融合。

**图像畸变矫正模块**（[[XMU-SG-SDD-UND-010]]）实现了[[XMU-SG-SRS-VideoProcessing]]视频处理能力需求中的预处理子能力，确保输入图像的几何准确性。

**目标检测模块**（[[XMU-SG-SDD-DET-006]]）实现了[[XMU-SG-SRS-ObjectDetection]]目标检测能力需求，使用深度学习模型实现船舶目标的检测和识别，满足目标识别准确率≥85%的需求指标。

#### 4.1.3.4 通信管理模块

**UDP通信模块**（[[XMU-SG-SDD-UDP-002]]）实现了[[XMU-SG-SRS-NetworkComm]]网络通信能力需求，负责与外部系统进行数据交换，包括接收控制指令和发送状态信息及检测结果。该模块对应上位机控制指令接口（[[XMU-SG-SRS-IF-SYS-001]]）和系统状态上报接口（[[XMU-SG-SRS-IF-SYS-002]]）。

**串口通信模块**（[[XMU-SG-SDD-SER-007]]）实现了可见光设备控制接口（[[XMU-SG-SRS-IF-HW-004]]）和红外设备控制接口（[[XMU-SG-SRS-IF-HW-005]]）需求，负责与硬件设备的直接通信。

#### 4.1.3.5 系统支撑模块

**视频编码模块**（[[XMU-SG-SDD-VID-011]]）实现了[[XMU-SG-SRS-VideoProcessing]]视频处理能力需求中的编码输出子能力，将处理后的图像编码成视频流，支持H.264编码视频流输出接口（[[XMU-SG-SRS-IF-NET-001]]）需求。

**系统参数管理模块**（[[XMU-SG-SDD-PAR-008]]）负责读取和管理系统的各项参数设置，支持系统的适应性需求。

**日志记录模块**（[[XMU-SG-SDD-LOG-009]]）记录系统的运行日志和故障信息，便于后续的分析和维护，支持[[XMU-SG-SRS-REQ-SAFETY-001]]输入数据校验等安全性需求的实现。

**时间同步模块**（[[XMU-SG-SDD-TIM-012]]）确保各个模块之间的时间一致性，支持[[XMU-SG-SRS-QR-RT-001]]融合实时性和[[XMU-SG-SRS-QR-RT-002]]识别实时性需求，这对于多源数据的融合至关重要。

### 4.1.4 每个软件单元的开发状态/类型

所有软件单元都是为本项目专门开发的新研软件，没有重用已有的设计或软件。每个软件单元都被分配了具体的功能，这些单元共同实现了模块的完整功能。这种模块化的设计有效提高了系统的可维护性和可扩展性。

#### 4.1.5.2 使用数据基于的假设和条件

硬件资源使用数据基于以下假设和条件：

- **典型用法**：系统正常运行状态下，处理频率为25Hz的图像数据和10Hz的点云数据，满足[[XMU-SG-SRS-QR-RT-001]]融合实时性需求
- **最坏情况用法**：同时处理三光源最大数据量，执行复杂目标检测算法，进行实时数据融合和编码输出，满足[[XMU-SG-SRS-QR-REL-002]]识别准确性≥85%的需求要求
- **特定事件假定**：系统启动初始化阶段和算法模型加载阶段的峰值资源需求

#### 4.1.5.3 影响使用的特殊考虑

影响计算机硬件资源使用的特殊考虑包括：

- **NPU加速器影响**：专用神经网络处理单元承担深度学习推理计算，降低CPU计算负荷，支持[[XMU-SG-SRS-ObjectDetection]]目标检测能力需求
- **RGA硬件加速库影响**：图像缩放、格式转换等操作通过硬件加速实现，减少CPU资源占用，支持[[XMU-SG-SRS-DataFusion]]数据融合能力需求
- **多处理器使用情况**：采用多线程并行处理架构，充分利用多核CPU资源，支持[[XMU-SG-SRS-QR-RT-001]][[XMU-SG-SRS-QR-RT-002]]实时性需求
- **操作系统开销影响**：考虑Linux系统内核调度和内存管理开销对整体性能的影响

### 4.2.1 软件单元间动态关系概述

三光融合与识别处理软件各部件执行方案见图4。该执行方案描述了软件单元间的执行控制流程、数据流和动态控制序列。

**执行控制流程说明：** 系统执行按照以下顺序进行：

1. 初始化串口[P1] - 工作流控制模块启动串口通信模块
2. 初始化UDP[P1] - 工作流控制模块启动UDP通信模块
3. 启动图像采集[P2] - 启动图像采集处理模块
4. 启动点云处理[P2] - 启动点云处理模块
5. 图像传输[P3] - 图像采集处理模块向目标检测模块传输图像数据
6. 点云数据传输[P3] - 点云处理模块向数据融合模块传输点云数据
7. 目标检测结果传输[P3] - 目标检测模块向数据融合模块传输检测结果
8. 融合结果传输[P4] - 数据融合模块向工作流控制模块传输融合结果
9. 状态更新[P4] - 工作流控制模块内部状态更新
10. 发送状态信息[P5] - 工作流控制模块通过串口通信模块发送状态信息

**数据流说明：** 数据流向为：图像采集处理模块→目标检测模块→数据融合模块→工作流控制模块→串口通信模块；点云处理模块→数据融合模块→工作流控制模块→串口通信模块

**单元间优先关系：**

- [P1]=最高优先级：初始化串口、初始化UDP
- [P2]=高优先级：启动图像采集、启动点云处理
- [P3]=正常优先级：图像传输、点云数据传输、目标检测结果传输
- [P4]=低优先级：融合结果传输、状态更新
- [P5]=最低优先级：发送状态信息

### 4.2.2 异常处理机制

**异常处理：** 系统设计了异常处理机制，当出现异常时执行重新初始化流程，并通过串口通信模块发送错误状态信息。

### 4.2.3 动态控制序列

**时序/排序关系：** 各软件单元按照优先级顺序执行，高优先级任务优先于低优先级任务。初始化阶段完成后，系统进入[初始化成功]状态，随后进入[并发执行]状态进行数据处理。

**并发执行说明：** 在[并发执行]状态下，图像采集处理、点云处理、目标检测等模块可并发运行，通过优先级机制协调执行顺序。

## 7.1 技术背景

### 7.1.1 三光融合技术背景

三光融合技术是指将可见光、红外热成像和激光雷达三种不同波段的传感器数据进行时空配准和信息融合的技术。该技术能够充分发挥各传感器的优势，克服单一传感器在特定环境下的局限性，实现全天候、高精度的目标检测识别。

### 7.1.2 海上目标检测应用背景

海上环境具有复杂多变的特点，包括海况变化、光照条件差异、气象影响等。传统单一传感器难以在所有条件下保持稳定的检测性能。三光融合技术通过多模态信息互补，能够显著提高海上目标检测的准确性和可靠性。

### 7.1.3 边缘计算架构背景

本软件采用边缘计算架构，在靠近数据源的设备上进行实时处理，减少数据传输延迟，提高系统响应速度。该架构特别适合对实时性要求较高的海上监控应用场景。

## 7.6 算法公式说明

### 7.6.1 图像融合权重公式

可见光与红外图像融合采用加权平均方法：

```
I_fusion(x,y) = α × I_visible(x,y) + β × I_infrared(x,y)
```

其中：

- `I_fusion(x,y)`：融合后图像像素值
- `I_visible(x,y)`：可见光图像像素值
- `I_infrared(x,y)`：红外图像像素值
- `α = 0.4`：可见光权重系数
- `β = 0.4`：红外权重系数
- `α + β = 0.8`：权重和小于1，为激光雷达数据保留空间

### 7.6.2 坐标变换公式

激光雷达点云到图像坐标的变换：

```
[u, v, 1]^T = K × [R|t] × [X, Y, Z, 1]^T
```

其中：

- `[u, v]`：图像像素坐标
- `[X, Y, Z]`：激光雷达点云坐标
- `K`：相机内参矩阵（3×3）
- `[R|t]`：旋转平移矩阵（3×4）

### 7.6.3 检测置信度计算

目标检测置信度通过以下公式计算：

```
Confidence = P(Object) × IoU(pred, truth)
```

其中：

- `P(Object)`：目标存在概率
- `IoU`：预测框与真实框的交并比
- `pred`：预测边界框
- `truth`：真实边界框
